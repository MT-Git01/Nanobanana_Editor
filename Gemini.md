# Gemini.md: インフォグラフィック・テキスト編集可能化アプリ設計仕様書

## 1. 概要 (Overview)
本アプリケーションは、Nanobanana2等で作成されたインフォグラフィック画像（またはPDF）を解析し、「文字の削除・背景補完」と「編集可能なテキストオブジェクトへの変換」を同時に行うことで、GoogleスライドやMicrosoft PowerPointで再編集可能なファイルを自動生成するシステムである。

## 2. 開発スタック (Tech Stack)
- **プラットフォーム:** Antigravity
- **フロントエンド:** Streamlit
- **バックエンド:** Python 3.10+
- **OCR・座標検出エンジン:** Google Cloud Vision API (推奨)
  - 理由: 文字認識率が極めて高く、各文字・ブロックの4頂点座標（Bounding Box）を高精度に取得できるため。
- **画像処理（文字消去）:** OpenCV (Inpaintingロジック) 
- **Officeファイル生成:** `python-pptx`

## 3. 機能要件 (Functional Requirements)

### 3.1. OCR・文字位置・内容の永続化
1. アップロードされた画像から、文字の「テキスト内容」と「配置座標（X, Y, Width, Height）」を抽出。
2. 抽出データをシリアライズし、Streamlitの `st.session_state` または一時データストアに保管。

### 3.2. 自動インペインティング（文字消去と背景復元）
1. OCRが検出した文字のBounding Boxを基にマスク画像を生成（文字より数ピクセル外側まで自動拡張）。
2. OpenCVの `cv2.inpaint` 処理を施し、元のデザイン（単色・グラデーション等）を壊さずに文字だけを綺麗に消去した「背景専用画像」を出力。

### 3.3. フォントの再現性と代替提案機能（ご要望に基づく追加）
1. 画像内のオリジナルフォントの完全一致が困難な場合に備え、システム内部に**「代替フォントマップ（Gothic系、Mincho系、Round系、Design系）」**を定義。
2. OCRが検出した文字特性から最も近いスタイルを自動選択し、存在しない場合は類似の標準フォント（例: Noto Sans, Arial, MS Gothic等）を提案。
3. StreamlitのUI上にブロックごとのフォント選択ドロップダウン（`st.selectbox`）を設け、ユーザーが任意の代替フォントへ手動変更できるようにする。

### 3.4. レイアウト崩れ防止・フォントサイズ調整機能（ご要望に基づく追加）
1. **自動スケーリング:** ユーザーのテキスト編集により文字数が増加した場合、元の配置枠（$W_{box}$）をテキスト幅（$W_{text}$）が超えないよう、以下の数式を用いてフォントサイズを自動縮小。
   $$S_{new} = S_{init} \times \min\left(1.0, \frac{W_{box}}{W_{text}}\right)$$
2. **手動微調整 UI:** 各テキストブロックの横に、フォントサイズを1pt単位で細かく変更できるスライダー（`st.slider`）またはカウンターボタンを配置。

### 3.5. Googleスライド / MS-PowerPoint 互換出力
1. `python-pptx`ライブラリを使用し、新規スライドの背景に「文字消去済みの背景画像」を全面配置。
2. 保管・編集されたテキストデータを、ピクセル-インチ変換ロジックを介して、元の位置（座標）へPowerPointの「テキストボックスオブジェクト」として動的に追加。
3. 生成された `.pptx` ファイルは、PowerPointでそのまま編集できるほか、Googleドライブにアップロードすることで**Googleスライドとして100%編集可能な状態でインポート可能**となる。

## 4. データフロー (Data Flow)
1. **[User]** 画像/PDF（例: MF説明.pdf）をStreamlit UIにアップロード。
2. **[Backend]** Cloud Vision APIで文字内容・座標を取得 ＆ OpenCVで背景画像を生成。
3. **[UI]** 画面上に背景画像を表示し、右側に編集用テキストフォーム、フォント選択、サイズ調整スライダーを動的に生成。
4. **[User]** フォーム上で文字を修正、またはフォントサイズを微調整。
5. **[Backend]** 編集確定後、`python-pptx`が走り、背景画像とテキストオブジェクトを合成した `.pptx` を生成。
6. **[UI]** ユーザーへダウンロードリンクを提示。
